# Face Detector — Real-Time Three-Person Desktop Alert

Local face-recognition alert system for a fixed webcam. Detects, recognises,
and sends desktop popups when one of three enrolled people appears.

## Requirements

- **Python 3.11+** (tested on 3.12)
- **CPU only** — no CUDA, no GPU
- **Ubuntu** (for `notify-send` notifications)

## Phase 1 Setup

### 1. Create a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 2. Install dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 3. Install Ubuntu notification support (if missing)

```bash
sudo apt update && sudo apt install libnotify-bin
```

Test notifications:

```bash
notify-send "Person Alert" "Notification system is working."
```

### 4. Test the camera

```bash
python -m scripts.test_camera --help
python -m scripts.test_camera                      # 10-second preview
python -m scripts.test_camera --source 0 --no-preview
python -m scripts.test_camera --timeout 5 --no-preview
```

Press `Q` or `q` to quit the preview window. The script also exits after
the default 10-second timeout.

### 5. Test notifications

```bash
python -m scripts.test_notification                 # check availability
python -m scripts.test_notification --send          # send a test popup
```

### 6. Test the face model

```bash
python -m scripts.test_face_model                   # verify model loads
python -m scripts.test_face_model --image /path/to/image.jpg
```

The first run may download model assets (~150–300 MB) from the InsightFace
model hub. This download happens automatically and requires an internet
connection.

### 7. Run unit tests

```bash
python -m pytest tests/ -v
```

Hardware-dependent tests are marked `@pytest.mark.manual` and are skipped
by default. To run them:

```bash
python -m pytest tests/ -v -m manual
```

## Configuration

See `config.yaml` for all settings. The configuration is loaded and validated
at startup by `src/config.py`.

## Project Structure

```
├── config.yaml              # Runtime configuration
├── requirements.txt         # Python dependencies
├── README.md
├── src/
│   ├── __init__.py
│   ├── config.py            # Typed config loading and validation
│   └── face_engine.py       # InsightFace detection + embedding
├── scripts/
│   ├── __init__.py
│   ├── test_camera.py       # Webcam preview and FPS test
│   ├── test_notification.py # notify-send availability check
│   └── test_face_model.py   # InsightFace model smoke test
├── tests/
│   ├── __init__.py
│   ├── test_config.py       # Config parsing and validation
│   ├── test_camera.py       # Camera cleanup behaviour
│   └── test_notification.py # Notification command construction
└── data/
    ├── reference_images/    # Per-person face images (user-provided)
    ├── embeddings/          # known_faces.npz (generated by enrollment)
    ├── logs/                # Event CSV files
    ├── snapshots/           # Alert snapshots
    └── calibration/         # Threshold calibration images
```

## Privacy

- Reference images, embeddings, logs, and snapshots are never committed
  (see `.gitignore`).
- Face embeddings are biometric identifiers. Keep `data/embeddings/` local.
